## 2. НАЗНАЧЕНИЕ И ЦЕЛИ СОЗДАНИЯ СИСТЕМЫ

### 2.1. Назначение системы
Система предназначена для автоматизации учета личных финансовых операций физических лиц. Обеспечивает ведение учета доходов и расходов, категоризацию операций, расчет баланса и формирование базовых отчетов.

### 2.2. Цели создания системы
- Снижение времени на учет финансовых операций с 30 минут в день до 5 минут (повышение эффективности на 83%)
- Повышение точности финансового учета за счет исключения ручных ошибок
- Обеспечение наглядного представления финансовой статистики
- Предоставление инструментов для анализа трат по категориям
- Упрощение процесса планирования личного бюджета

## 3. ХАРАКТЕРИСТИКИ ОБЪЕКТОВ АВТОМАТИЗАЦИИ

### 3.1. Описание объекта автоматизации
Объектом автоматизации является процесс ведения личного финансового учета, включающий:
- Регистрацию финансовых операций (доходы и расходы)
- Категоризацию операций по типам трат
- Расчет текущего финансового баланса
- Формирование отчетов за различные периоды

### 3.2. Текущее состояние и проблемы
**Текущее состояние:** Ведение учета в тетрадях, Excel-таблицах или отсутствие систематического учета

**Проблемы, решаемые автоматизацией:**
- Потеря данных об операциях из-за забывчивости
- Ошибки при ручном подсчете баланса
- Отсутствие анализа структуры трат
- Сложность формирования отчетов за период
- Невозможность быстро оценить финансовое состояние

### 4.3. Требования к видам обеспечения

#### 4.3.1. Математическое обеспечение
- **Алгоритм расчета баланса:** Баланс = Σ(доходы) - Σ(расходы)
- **Округление сумм:** до 2 знаков после запятой (копейки)
- **Валидация дат:** проверка корректности формата YYYY-MM-DD
- **Сортировка операций:** по дате (убывание) как основной критерий

#### 4.3.2. Программное обеспечение
- **Операционная система:** Windows 10+, Linux (Ubuntu 18.04+), macOS 10.15+
- **Среда выполнения:** Python 3.8 или выше
- **База данных:** SQLite 3.31+
- **GUI библиотека:** Tkinter (входит в Python)
- **Дополнительные библиотеки:** SQLAlchemy 1.4+, matplotlib 3.5+

#### 4.3.3. Техническое обеспечение
**Минимальные требования:**
- **Процессор:** Intel Core 2 Duo 2.0 GHz или аналогичный
- **Оперативная память:** 2 ГБ
- **Свободное место на диске:** 100 МБ
- **Разрешение экрана:** 1024x768

**Рекомендуемые требования:**
- **Процессор:** Intel Core i3 или выше
- **Оперативная память:** 4 ГБ
- **Свободное место на диске:** 500 МБ
- **Разрешение экрана:** 1366x768 или выше

#### 4.3.4. Информационное обеспечение

**Структура базы данных:**

```sql
-- Таблица категорий
CREATE TABLE categories (
    id INTEGER PRIMARY KEY,
    name TEXT UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Таблица операций
CREATE TABLE operations (
    id INTEGER PRIMARY KEY,
    amount DECIMAL(10,2) NOT NULL,
    type TEXT CHECK (type IN ('income', 'expense')),
    category_id INTEGER REFERENCES categories(id),
    description TEXT,
    date TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Форматы файлов:**
- База данных: SQLite (.db)
- Экспорт: CSV (UTF-8)
- Конфигурация: INI
- Логи: текстовые файлы (.log)

#### 4.3.5. Лингвистическое обеспечение
- **Язык интерфейса:** Русский
- **Кодировка:** UTF-8
- **Денежные суммы:** российские рубли, формат "1 234,56 ₽"
- **Даты:** формат ДД.ММ.ГГГГ для отображения, ГГГГ-ММ-ДД для хранения
- **Терминология:** стандартная финансовая терминология
